# Feedback and support

Power Apps code apps use fully supported Microsoft technologies, such as Power Platform services, SDKs, CLI tooling, and supported Azure dependencies. Code apps also include developer-facing tooling and samples that evolve rapidly.
This article explains when to use your standard channel for Microsoft Support versus when to file a GitHub issue so feedback and issues reach the right channel and can be addressed effectively.

If you face issues with:

- **Power Apps SDK or CLI behavior that doesn't match the documentation:** Use your standard channel for Microsoft Support.
- **App behavior that regresses without making changes:** Use your standard channel for Microsoft Support.
- **Sharing feedback, seeking guidance, or reporting other issues:** [Power Apps code apps GitHub repository issues](https://github.com/microsoft/PowerAppsCodeApps/issues). This feedback includes seeking guidance for functionality that isn't explicitly documented on learn.microsoft.com or the official code apps GitHub repository.
