# Feedback and support

Power Apps code apps are built on fully supported Microsoft technologies (such as Power Platform services, SDKs, CLI tooling, and supported Azure dependencies), code apps also includes developer-facing tooling and samples that evolve rapidly.
This article explains when to use your standard channel for Microsoft Support versus when to file a GitHub issue so feedback and issues reach the right channel and can be addressed effectively.

If you face issues with:
- <b>Power Apps SDK or CLI behavior does not match documentation:</b> use your standard channel for Microsoft Support
- <b>App behavior regresses without making changes:</b> use your standard channel for Microsoft Support
- <b>You want to share feedback or seek guidance: report your issue here:</b> [Power Apps code apps GitHub repository issues](https://github.com/microsoft/PowerAppsCodeApps/issues). This includes seeking guidance for functionality that is not explicitly documented on learn.microsoft.com or the official code apps GitHub repository.
