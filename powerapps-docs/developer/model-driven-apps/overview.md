# Model-driven apps developer guide

Model-driven apps are primarily a no-code or low-code component-focused approach to app development. Developers add value to model-driven apps by extending the application with custom code, components, and integrations. Before you start writing code, [learn how to build model-driven apps](../../maker/model-driven-apps/model-driven-app-components.md) and explore what options you can apply without code.

> [!NOTE]
> Model-driven apps connect to Microsoft Dataverse. For information about how developers can add value at the service level, see [Dataverse Developer Overview](../data-platform/overview.md).
>
> This section refers only to extensions that developers can make to improve the experience for users of model-driven apps. 

If you're new to Dataverse applications, the topics in this section provide a high-level overview of the important concepts to help developers get started working with model-driven apps.
